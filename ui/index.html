Commands

Slash commands that trigger coordinated multi-agent workflows. Invoke them directly in OpenCode.

/ build
Analyzes your project description, selects the right workflow and agents, and produces a structured implementation plan with full architecture.
→ build-application workflow
/ architect
Designs or reviews system architecture. Produces component diagrams, ADRs, scalability analysis, and open questions.
→ principal-engineer
/ debug
Systematic bug investigation. From symptom to root cause to targeted fix with a regression test and documentation.
→ bug-investigation workflow
/ review
Senior-level code review across correctness, security, performance, and maintainability. Provides severity-rated findings.
→ principal-engineer + security
/ secure
Full security audit with threat modelling, OWASP checks, dependency scanning, and a CVSS-rated findings report.
→ security-audit workflow
/ optimize
Measurement-driven performance optimization. Covers algorithms, DB queries, caching strategy, and infrastructure tuning.
→ performance-audit workflow

Quick Start

Get up and running with GOD-OPENCODE in under 5 minutes.

1. Install
# Clone the repo git clone https://github.com/your-org/god-opencode C:\AI\GOD-OPENCODE cd C:\AI\GOD-OPENCODE # Run the one-command installer .\god-install.ps1 # Verify installation .\god-health.ps1
2. Use slash commands in OpenCode
# Build a new application /build a FastAPI REST API with JWT auth, PostgreSQL, and Alembic migrations # Debug a problem /debug users are getting duplicate emails on registration in auth_service.py # Security audit /secure review the authentication flow in routes/auth.py # Architecture review /architect should I use PostgreSQL or MongoDB for a multi-tenant SaaS? # Code review /review [paste code here] # Performance optimization /optimize the user search endpoint is slow at 1000+ users
3. Invoke agents directly
# Direct agent invocation As the database-architect, design a schema for a multi-tenant SaaS application where each tenant's data must be isolated. As the security-engineer, audit this JWT implementation: [paste code] As the ai-engineer, design a RAG pipeline for a customer support chatbot with 50k support articles.
4. Scan a project
# Scan any project and get an implementation plan .\tools\project-scan.ps1 -TargetPath "C:\path\to\your\project" # The scanner will: # 1. Detect primary languages # 2. Identify project type (api, fullstack, frontend-app, etc.) # 3. Recommend the right workflow and agents # 4. Generate a prioritized improvement list # 5. Save the plan to memory/adr-scan-{project}.md
